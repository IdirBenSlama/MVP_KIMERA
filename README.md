## 🚀 Quick Start

### Installation

```bash
# Clone repository
git clone https://github.com/your-org/kimera-swm.git
cd kimera-swm

# Create virtual environment
python -m venv .venv
source .venv/Scripts/activate  # Windows
# source .venv/bin/activate    # macOS/Linux

# Install dependencies
pip install -r requirements.txt

# Start system
python run_kimera.py
```

### First API Call

```bash
# Create a semantic geoid
curl -X POST http://localhost:8000/geoids \
  -H "Content-Type: application/json" \
  -d '{
    "semantic_features": {"concept": 0.8, "intensity": 0.6},
    "symbolic_content": {"type": "example"},
    "metadata": {"source": "quickstart"}
  }'
```

```bash
# Check system health
```

### Example Usage

```python
import requests

# Create geoid
response = requests.post('http://localhost:8000/geoids', json={
    "semantic_features": {"concept": 0.8, "polarity": 0.6},

geoid = response.json()
print(f"Created geoid: {geoid['geoid_id']}")

print(f"Created {result['scars_created']} scars")

print(f"Self-analysis geoids: {analysis['geoids_created']}")
print(f"Self-contradictions: {analysis['contradictions_detected']}")

# Trigger adaptive fine-tuning (NEW)
response = requests.post('http://localhost:8000/system/optimize')
```

## 🧐 Troubleshooting

### Python Virtual Environment Issues

The system is designed to run in an isolated Python virtual environment (`.venv`). If you encounter `ModuleNotFoundError` errors:

**Solution:**
1. **Activate the environment:**
   ```bash
   # For Windows (Git Bash or similar)
   source .venv/Scripts/activate
   
   # For macOS/Linux
   source .venv/bin/activate
   ```
2. **Verify the interpreter:**
   ```bash
   which python  # Should point to .venv directory
   which pip     # Should point to .venv directory
   ```

### Dependency Import Errors

If the server fails to start with import errors:

**Solution:**
1. Ensure virtual environment is active
2. Force reinstall problematic dependencies:
   ```bash
   pip install --upgrade --force-reinstall torch torchvision transformers
   ```

### Meta-Cognitive Processing Issues

If self-analysis features fail:

**Solution:**
1. Ensure all documentation files are present
2. Check API server is running on correct port
3. Verify sufficient memory for embedding processing
4. Review logs for specific error messages
